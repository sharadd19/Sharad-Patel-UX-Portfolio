NUAR

Designing and implementing a page to manage users and organisations using the NUAR platform.

Tools used: Figma, C#, JavaScript, HTML, CSS, SQL

Introduction

NUAR is a web application that allows users to view assets such as fiber optic lines, gas lines, water lines, etc. underground. This is so that when construction workers dig underground, they don’t dig into one of these lines, cutting it and potentially disruptions many homes.

There are two sides to NUAR. The Web Portal, which contains the user interface for the map and it’s assets, and the Admin Portal which contains the functionality for being able to manage everything. I worked on the Admin Portal in this project.

Problem

There was no user interface for creating contracts between organisations. A contract is an agreement for work between two organisations.

Previously, you couldn’t create any contracts. There was no UI UX team for the Admin Portal since it was a very small part of the overall project, therefore I was given the freedom to design this page independently. The only constraint was to adhere to the Government Design Service (GDS) styling.

Goal

Create a page which allows the admins of organisation to create contracts with other organisations so that they could expand their work.

Impact

The impact of this work meant that there was increased visibility of areas in the UK, which allowed a more comprehensive overview of how assets are distributed underground. The more contracts formed, the more information the map receives, and the more detailed our view becomes.

User Personas

There are two personas. We have the Super User, and we have the Admin.

  • Super User - These are the developers from Atkins Realis, who control the entire system. They have permissions to do everything
  • Admins - These are the users of the organisation who have been nominated to be Admins. They have permissions to only do certain things, listed in the Early Ideation section.

    Testing

    This was handed-off to user research where they did some usability testing with users and brought some feedback.

    One of the pieces of feedback were that users found that their organisation type was irrelevant information and it takes up a large portion of space in the view. The other was that contract was a confusing term, so we replaced it with ‘Supply Chain’.

    In addition to this, the Create Contract page had a problem. A user could type their organisation in slightly differently or with capital letters and create duplicate accounts. This was resolved by having a drop down list so that users could select their organisation (see below). This required them to be onboarded and invited first.

    Development

    Upon onboarding more users, we found that the dropdown lists when creating organisations were incredibly large and required a lot of scrolling to find your organisation. An idea I had to make them more scrollable was to somehow implement search functionality.

    This could be done with:

  • A table view listing all the organisations like how it’s done in the manage contracts page
  • A searchable drop down list

    • The former would be slow and clunky whereas the latter as it would be more intuitive from a user’s point of view.
